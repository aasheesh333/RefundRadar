**Goal:** Keep real SMS inbox import while hardening SMS consent, notification correctness, reminder dismissal, premium template access, and production verification.

**Architecture:** Keep the current Flutter/Riverpod architecture and Android MethodChannel SMS bridge. Make small, focused changes in the existing SMS, notification, reminder, template, l10n, and docs files; avoid new abstractions unless a task explicitly defines a small helper.

**Tech Stack:** Flutter, Dart, Riverpod, GoRouter, permission_handler, flutter_local_notifications, Kotlin Android `Telephony.Sms.Inbox`, Firebase/Firestore repositories, custom `AppLocalizations` map + ARB files.

## Global Constraints

- Keep `READ_SMS` and real SMS inbox import; user will declare SMS permission in Play Console.
- SMS import remains on-device: native Android reads inbox rows, Dart filters likely financial/refund messages, and parsing does not upload raw SMS for import.
- Permission copy must not claim the app can technically access only financial SMS. It must state that Android grants inbox access and RefundRadar scans locally for relevant messages.
- Manual paste remains first-class. Permission denial must not block dispute creation.
- Notification IDs must be deterministic from stable string input using app-owned code, not Dart `String.hashCode`.
- Reminder lifecycle changes must update both persisted reminder state and local scheduled notifications when possible.
- Premium access checks must be consistent across list, preview, and copy actions.
- UI changes should use existing components, localization patterns, and `AppThemeColors`; no broad redesign.
- Verification: run `/home/ubuntu/flutter-sdk/bin/flutter analyze` and targeted tests locally; use GitHub Actions for full Analyze + Test after push.
- Do not run a full local Flutter test suite or APK build unless explicitly approved, because this host has prior OOM/crash risk.

```dart
  static int scheduledIdFor(String reminderId) {
    var hash = 0x811c9dc5;
    for (final byte in reminderId.codeUnits) {
      hash ^= byte;
      hash = (hash * 0x01000193) & 0xFFFFFFFF;
    }
    return hash & 0x7FFFFFFF;
  }

  static int cancelIdFor(String reminderId) {
    return scheduledIdFor(reminderId);
  }
```

```dart
/// Dismiss one reminder and cancel its matching local notification.
/// Firestore dismissal is the source of truth. Local notification cancellation
/// is best-effort after the row is marked dismissed, so a platform failure does
/// not resurrect a dismissed reminder in the UI.
Future<void> dismissReminderAndCancelNotification(
  dynamic ref,
  String uid,
  String reminderId,
) async {
  final repo = ref.read(reminderRepositoryProvider) as ReminderRepository;
  await repo.dismiss(uid, reminderId);
  try {
    final notifications = ref.read(notificationServiceProvider) as NotificationService;
    await notifications.cancelForReminder(reminderId);
  } catch (e, st) {
    debugPrint('cancelForReminder failed for $reminderId: $e\n$st');
  }
}
```
